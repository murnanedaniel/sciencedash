## Research drifts

For a capable, curious researcher the hard part is no longer having ideas, tools,
data, or compute. It's **consolidation** — turning exploration into finished,
visible output. Projects wander off and leave nothing behind; the good ones often
get pushed over the line by someone else. ScienceDash exists to change the *shape*
of that walk: keep exploring, but leave a durable artifact at every step.

## A research operating system

ScienceDash structures research as four layers. Each turns the messy output of the
one above it into something more finished — and a daily ritual plus AI critical
review keep projects from quietly stalling.

## The loop: idea → paper

Declare a metric and a compute budget *before* you run. Log runs against
hypotheses. When a hypothesis lands, spawn a paper skeleton and fill it from your
own runs. Every resolution, paper, and accepted edit writes to a decision log — so
you can see whether the walk is moving forward.

## Claude-native, not another agent

ScienceDash is a **cockpit and portfolio brain — not an agent runtime**. Your
Claude Code sessions do the work; ScienceDash holds the cross-cutting state no
single session can. One MCP substrate is read and written by every session alike:
dashboard buttons, your local terminal, and long-running workhorses on remote
compute. The axiom: *native Claude Code beats any harness.*

## Get it running

Local-first: it runs on `localhost`, your data is one SQLite file, and AI features
bill against your Claude subscription — no `ANTHROPIC_API_KEY`.

```bash
git clone https://github.com/murnanedaniel/sciencedash
cd sciencedash/web
npm install
cp .env.example .env       # generate the 4 auth values (see the file)
npx prisma migrate dev
npm run db:seed            # optional: sample projects so it's not empty
npm run build && npm run start
```

Then open [http://localhost:3000](http://localhost:3000). Full walkthrough in
[Getting started](/sciencedash/getting-started/).
