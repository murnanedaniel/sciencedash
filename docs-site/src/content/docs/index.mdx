## What it is

ScienceDash is a single-researcher "operating system" for going from an idea to
a paper. It keeps the random walk honest: declare what *winning* means before
you run, log runs against hypotheses, resolve those hypotheses into papers, and
watch a decision log that shows whether you're actually moving forward.

<CardGrid stagger>
  <Card title="Inner loop" icon="rocket">
    Projects → hypotheses → runs → metrics. Declare a primary metric and a
    compute budget up front; the meter goes amber when you blow past it.
  </Card>
  <Card title="Consolidation" icon="open-book">
    When a hypothesis lands, spawn a six-section paper skeleton, fill it from
    your runs with an AI first pass, polish per section, export `.tex`.
  </Card>
  <Card title="Workhorses" icon="puzzle">
    Dispatch Claude Code sessions onto remote compute (a GPU box, an HPC login
    node) over a small sync protocol, and watch their health from the sidebar.
  </Card>
  <Card title="On your terms" icon="approve-check">
    Local-first: one SQLite file, runs on <code>localhost</code>. AI features
    bill against your Claude Pro/Max subscription — no <code>ANTHROPIC_API_KEY</code>.
  </Card>
</CardGrid>

## Get it running

```bash
git clone https://github.com/murnanedaniel/sciencedash
cd sciencedash/web
npm install
cp .env.example .env       # generate the 4 auth values (see the file)
npx prisma migrate dev
npm run db:seed            # optional: sample projects so it's not empty
npm run build && npm run start
```

Then open [http://localhost:3000](http://localhost:3000). Full walkthrough in
[Getting started](/sciencedash/getting-started/).
